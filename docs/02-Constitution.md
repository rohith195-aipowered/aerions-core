# Aerions Constitution

This document defines the permanent principles of Aerions.

Technology will evolve.

Models will change.

These principles should remain stable.

---

# Principle 1

Objectives over Prompts.

Users should describe outcomes.

Aerions is responsible for determining execution.

---

# Principle 2

Models are replaceable.

No component inside Aerions should permanently depend on a specific AI model or provider.

Every model is a plugin.

---

# Principle 3

Context before Intelligence.

No AI model should be invoked before Aerions gathers sufficient context.

Reasoning without context produces unreliable results.

---

# Principle 4

Planning before Execution.

Every objective should first become a structured execution plan.

Execution is the final step, not the first.

---

# Principle 5

Single Responsibility.

Every module inside Aerions must solve exactly one problem.

Complex behavior emerges from combining simple modules.

---

# Principle 6

Intelligence is composable.

Multiple models may cooperate on a single objective.

Aerions decides how intelligence is composed.

---

# Principle 7

Infrastructure over Applications.

Aerions is not an end-user application.

It is infrastructure that intelligent applications build upon.

---

# Principle 8

Memory belongs to the platform.

Applications should not manage intelligence independently.

Knowledge should be reusable across objectives.

---

# Principle 9

The best model is the one that accomplishes the objective.

Model popularity is irrelevant.

Capability determines selection.

---

# Principle 10

Architecture before Features.

Every new feature must strengthen the architecture.

Nothing should compromise the long-term vision for short-term convenience.
