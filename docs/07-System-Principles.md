# Aerions System Principles

## 1. Objectives over Prompts
Users express what they want to achieve.
Aerions determines how to achieve it.

## 2. Context before Action
Every objective begins with understanding.

## 3. Capabilities before Models
Aerions chooses the required capabilities first, then selects the best model.

## 4. Models are Replaceable
No service depends on a specific model provider.

## 5. Verification is Mandatory
Every execution should be evaluated before the result is returned.

## 6. Modular by Default
Every component has one responsibility.

## 7. Build for the Next Decade
Short-term convenience must never compromise long-term architecture.
