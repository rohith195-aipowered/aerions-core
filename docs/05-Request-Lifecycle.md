# Request Lifecycle

Every request inside Aerions follows the same lifecycle.

---

## Stage 1

Receive Objective

↓

Convert natural language into a structured objective.

---

## Stage 2

Gather Context

↓

Collect every relevant piece of information.

Examples include:

- Website
- Images
- Documents
- APIs
- Internal Memory
- Previous Executions

---

## Stage 3

Planning

↓

Create an execution strategy.

Execution does not begin here.

---

## Stage 4

Capability Analysis

↓

Determine whether software alone can accomplish the task.

If YES

↓

Software Executor

If NO

↓

AI Router

---

## Stage 5

Execution

↓

Execute every planned step.

Execution may involve software, AI, or both.

---

## Stage 6

Evaluation

↓

Determine whether the objective has been achieved.

Possible outcomes:

- Success
- Retry
- Request More Context
- Ask User
- Change Provider

---

## Stage 7

Memory

↓

Persist reusable knowledge for future objectives.

---

## Stage 8

Return Response

↓

Deliver the final validated result to the user.
